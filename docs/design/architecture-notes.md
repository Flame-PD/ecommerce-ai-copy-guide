# 架构笔记

## 总体思路

项目采用前后端分离结构。前端负责演示、输入收集和结果展示；后端负责 API 契约、请求校验、AI 能力编排、数据读写和缓存策略。初始化阶段先用 deterministic Mock Service 替代真实 AI，保证接口和页面能稳定迭代。

## 规划组件

- Frontend：Vue 3 + Vite，提供 H5 导购页和后续管理后台入口。
- Web API：Flask，负责健康检查、能力清单和业务 API。
- Schema Layer：Pydantic，请求校验和后续响应结构约束。
- AI Service：封装文案生成、导购推荐、情感分析、直播脚本生成，当前为 Mock，后续接入真实模型。
- PostgreSQL：保存商品、评论、生成结果、用户会话和推荐记录。
- Redis：缓存热点商品、推荐结果、短期会话上下文和生成结果。
- Crawler：后续使用 Scrapy 抓取商品基础信息和评论样本。

## 数据流

1. 用户在前端输入商品、评论或导购需求。
2. 前端调用 Flask API。
3. API 使用 Pydantic 校验请求。
4. Service 层生成结果；初始化阶段使用模板规则，后续替换为模型调用。
5. 后续版本将把生成结果、评论分析结果和推荐记录写入 PostgreSQL，并用 Redis 缓存热点查询。

## 当前边界

- 不接真实 AI Provider。
- 不创建数据库迁移。
- 不实现用户登录、权限和后台管理。
- 不实现真实爬虫采集。
- 保留 Docker、配置项和目录结构，为下一阶段接入留接口。
